# Presentation Prep Checklist

A prioritised, tickable list to get from here to show-ready. Ordered by what unblocks
the most. Estimated total prep time: **one full rehearsal day + ~3 hours of setup.**

---

## A. Decisions to lock first (15 min — do today)

- [ ] **Confirm the fund's name** — deck currently reads *AllWeather Capital*; your brief
  said *AllWeather Capital*. Tell me which and I'll change it everywhere.
- [ ] **Confirm the 85% Trayport figure** matches Trayport's current approved external
  wording (public sources say "over 85% of European utilities' derivatives trading").
- [ ] **Add your UK retail-energy / Automated Trading detail** to slide 2 (I left this to
  you — didn't want to assert a number I couldn't verify).
- [ ] **Pick the Demo A company:** Clicks vs Dis-Chem comparison (recommended) or a single
  name. The fallback is already written for the comparison.

## B. Content finalisation (1–2 hours)

- [ ] Read the deck end-to-end in **Presenter View** so you see the notes pane.
- [ ] Read `presenter-script.md` once, out loud, for the narrative arc.
- [ ] Tighten any slide whose wording isn't *yours* — make the voice yours.
- [ ] Decide which backups you might need (Copilot deep-dive, determinism, security, cost)
  and bookmark them.
- [ ] Sense-check every number on slides 2 and 19 against source.

## C. Demo rehearsal (the highest-risk part — 2–3 hours)

- [ ] **Test Demo A on the actual presentation machine, same week** (not just your dev box).
- [ ] Time Demo A end-to-end so you know it comfortably finishes within ~20 min.
- [ ] **Regenerate the Demo A fallback** so the figures are current (`fallback/`), and take
  screenshots of it mid-run as a backup-to-the-backup.
- [ ] **Rehearse Demo B (mining build) twice with a timer. Hard cap 6 minutes.**
- [ ] Confirm the **fallback mining skill** triggers, and the **sample Excel** opens.
- [ ] Practise the recovery move: if a demo stalls, open the fallback file and narrate it as
  "here's one it prepared earlier." Never debug on stage.
- [ ] Have the **mining prompt** and **Demo A prompt** ready to paste (clipboard / notes
  file) — don't type them live.

## D. Tech & AV setup (~1 hour)

- [ ] Claude desktop app + Cowork signed in on the presentation machine.
- [ ] **Claude in Chrome extension** installed and connected (needed if any IR-site browsing
  is shown; not needed for the local-only Demo A).
- [ ] File / web permissions pre-approved so you don't hit prompts mid-demo.
- [ ] Screen layout tested: slides + Cowork side-by-side, or a fast app-switch you've drilled.
- [ ] **Font size up** in Cowork; output legible from the back of the room.
- [ ] **Notifications off** (Slack, Teams, email, OS) — full-screen / Do Not Disturb.
- [ ] Present from **`an-all-weather-analyst-v2.pptx`** — close the original first.
- [ ] Export a **PDF of the deck** as an emergency fallback if PowerPoint misbehaves.

## E. Room & logistics

- [ ] Projector/screen resolution checked; deck is 16:9.
- [ ] Clicker / advance method tested.
- [ ] Wi-Fi checked — but **Demo A stays local** regardless (don't rely on venue Wi-Fi).
- [ ] Know who the **SharePoint/M365 org admin** is, in case the connector question comes up.
- [ ] Water within reach.

## F. The day-of pre-flight (15 min before)

- [ ] Reboot; close everything except slides, Cowork, and the fallback folder.
- [ ] Demo A prompt on the clipboard.
- [ ] Fallback folder open in a window you can reach without fumbling.
- [ ] Notifications off; Do Not Disturb on.
- [ ] Presenter View up; water poured; phone on silent.
- [ ] One slow breath. The promise is true — you've rehearsed it.

## G. After the session

- [ ] Send the two **how-to guides (PDF)** and the **roadmap one-pager** to attendees.
- [ ] Nominate the **two pilot analysts** (your ask #1).
- [ ] Book the **week-2 30-minute review** (your ask #3).
- [ ] Capture the questions you couldn't fully answer → follow-up note.
- [ ] Note which slide/demo landed best and worst, for next time.
